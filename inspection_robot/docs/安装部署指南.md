## 🖥️ 环境准备

### 1.1 系统要求检查

#### 1.1.1 操作系统
本软件支持以下操作系统：
- **Windows 10** (64位) - 推荐版本 1903 或更高
- **Windows 11** (64位)

#### 1.1.2 硬件配置
| 组件 | 最低配置 | 推荐配置 |
|------|---------|---------|
| 处理器 | Intel i5-6500 / AMD Ryzen 5 | Intel i7-10700 / AMD Ryzen 7 |
| 内存 | 8GB | 16GB 或更多 |
| 硬盘空间 | 2GB 可用空间 | 5GB SSD 可用空间 |
| 显卡 | 集成显卡 | 独立显卡 |
| 显示器 | 1920×1080 | 2560×1440 |

#### 1.1.3 检查方法
```
1. 查看系统信息：
   Win + R → 输入 msinfo32

2. 查看处理器和内存：
   Win + R → 输入 dxdiag

3. 查看硬盘空间：
   右键C盘 → 属性
```

### 1.2 Python环境安装

#### 1.2.1 下载Python
1. 访问Python官网：https://www.python.org/downloads/
2. 选择 Python 3.8.x 或更高版本（推荐3.9+）
3. 下载Windows安装包（64位）

#### 1.2.2 安装步骤
```
1. 运行安装程序
2. 勾选以下选项：
   ☑ Add Python to PATH（添加到环境变量）
   ☑ Install for all users（所有用户）

3. 点击 Install Now 安装
4. 安装完成后关闭
```

#### 1.2.3 验证安装
```cmd
# 打开命令提示符（Win + R → cmd）
# 输入以下命令：
python --version
# 显示：Python 3.x.x

pip --version
# 显示：pip xx.x.x
```

## 💿 软件安装

### 2.1 获取软件

#### 2.1.1 安装包内容
```
inspection_robot_v1.0.0/
├── inspection_robot/           # 主程序目录
│   ├── main.py                # 启动文件
│   ├── requirements.txt       # 依赖包列表
│   ├── modules/              # 功能模块
│   ├── data/                 # 数据目录
│   ├── assets/               # 资源文件
│   └── docs/                # 文档
├── README.md                # 使用说明
├── LICENSE                  # 版权声明
└── 安装说明.txt            # 本文档
```

#### 2.1.2 解压安装
1. 将安装包解压到目标位置（如 C:\Program Files\）
2. 建议路径不包含中文字符
3. 确保有写入权限

### 2.2 依赖包安装

#### 2.2.1 创建虚拟环境（推荐）
```cmd
# 进入项目目录
cd C:\Users\pc\inspection_robot

# 创建虚拟环境
python -m venv venv

# 激活虚拟环境
venv\Scripts\activate

# 激活后命令行前缀会显示 (venv)
```

#### 2.2.2 安装依赖
```cmd
# 方法一：使用requirements.txt
pip install -r requirements.txt

# 方法二：手动安装
pip install PyQt5==5.15.9
pip install pyqtgraph==0.13.3
pip install numpy==1.24.3
pip install scipy==1.11.1
pip install matplotlib==3.7.2
pip install pandas==2.0.3
pip install pyserial==3.5
pip install Pillow==10.0.0
```

#### 2.2.3 使用国内镜像（加速）
```cmd
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 2.3 验证安装

#### 2.3.1 检查依赖包
```cmd
# 激活虚拟环境后运行
python -c "import PyQt5, numpy, scipy, matplotlib"
# 无错误提示表示安装成功
```

#### 2.3.2 运行测试
```cmd
python test_modules.py
# 所有测试通过表示安装正确
```

## 🔌 硬件配置

### 4.1 串口设备安装

#### 4.1.1 驱动安装
```
1. 下载串口驱动：
   - CH340驱动：http://www.wch.cn/download/CH341SER_EXE.html
   - CP2102驱动：https://www.silabs.com/products/development-tools/software/usb-to-uart-bridge-vcp-drivers

2. 运行安装程序，完成安装

3. 重启电脑
```

#### 4.1.2 查看COM端口
```
方法一：设备管理器
1. 右键"此电脑" → 管理
2. 设备管理器 → 端口(COM和LPT)
3. 记录显示的COM口（如 COM3）

方法二：命令行
python -c "import serial.tools.list_ports; print(serial.tools.list_ports.comports())"
```

#### 4.1.3 连接测试
```cmd
# 测试串口连接
python -c "import serial; s=serial.Serial('COM3', 115200); print('OK'); s.close()"
```

### 4.2 样机连接

#### 4.2.1 连接步骤
```
1. 确保样机电源已关闭
2. 连接USB转串口线到电脑
3. 连接另一端到样机控制板
4. 接通样机电源
5. 打开软件 → 选择"样机测试与控制"标签
6. 点击"连接硬件" → 选择COM端口
7. 点击"连接"按钮
```

#### 4.2.2 通信参数
```
波特率：115200 bps
数据位：8
停止位：1
校验位：无
流控制：无
```

### 4.3 虚拟模式

#### 4.3.1 启用虚拟模式
```
1. 如无硬件连接，软件会自动进入虚拟模式
2. 状态栏显示：模式：虚拟模式
3. 虚拟模式下控制指令会被软件模拟
```

#### 4.3.2 虚拟模式特点
- 无需硬件即可测试
- 模拟机器人运动状态
- 记录虚拟数据用于分析

## ❓ 常见问题

### 5.1 安装问题

#### Q1：Python安装后命令行找不到
```
解决方案：
1. 手动添加Python到环境变量
   右键"此电脑" → 属性 → 高级系统设置 → 环境变量

2. 在"系统变量"中编辑Path：
   添加 Python安装路径（如 C:\Python39）
   添加 Scripts路径（如 C:\Python39\Scripts）

3. 重启命令提示符
```

#### Q2：pip安装失败或速度慢
```
解决方案：
1. 升级pip
   python -m pip install --upgrade pip

2. 使用国内镜像
   pip install -i https://pypi.tuna.tsinghua.edu.cn/simple 包名

3. 手动下载whl文件
   https://pypi.org/simple/ 包名/
   pip install 文件名.whl
```

#### Q3：依赖包版本冲突
```
解决方案：
1. 创建干净的虚拟环境
   python -m venv venv_clean
   venv_clean\Scripts\activate

2. 按顺序安装
   pip install -r requirements.txt

3. 如果仍有问题，尝试降低版本号
```

### 5.2 运行问题

#### Q4：启动后界面显示异常
```
可能原因：
1. 分辨率过低（小于1366×768）
2. 显卡驱动问题
3. PyQt5安装不完整

解决方案：
1. 调整系统缩放设置为100%
2. 更新显卡驱动
3. 重装PyQt5
   pip uninstall PyQt5
   pip install PyQt5==5.15.9
```

#### Q5：软件崩溃或无响应
```
可能原因：
1. 内存不足
2. 数据文件损坏
3. 第三方库冲突

解决方案：
1. 关闭其他程序释放内存
2. 删除 data/ 目录下所有文件
3. 检查错误日志（如有）
```

#### Q6：串口连接失败
```
可能原因：
1. 端口被占用
2. 驱动未安装
3. 硬件未连接

解决方案：
1. 检查设备管理器中端口状态
2. 关闭其他可能使用串口的软件
3. 更换USB端口
4. 重新安装驱动
```

### 5.3 权限问题

#### Q7：无法写入数据文件
```
解决方案：
1. 以管理员身份运行
   右键软件图标 → 以管理员身份运行

2. 修改文件夹权限
   右键data文件夹 → 属性 → 安全
   添加用户写权限

3. 安装到用户目录
   如 C:\Users\用户名\inspection_robot
```

## 🗑️ 卸载说明

### 6.1 软件卸载

#### 6.1.1 删除软件目录
```
1. 关闭软件
2. 删除软件安装目录
3. 删除桌面快捷方式
```

#### 6.1.2 清理虚拟环境
```cmd
# 进入软件目录
cd C:\Users\pc\inspection_robot

# 删除虚拟环境
venv\Scripts\deactivate
rmdir /s /q venv
```

### 6.2 清理系统

#### 6.2.1 删除环境变量
```
如需彻底清理：
1. 环境变量 → Path
2. 删除 Python 和 Scripts 路径
3. 重启电脑
```

#### 6.2.2 卸载Python（可选）
```
1. 控制面板 → 程序和功能
2. 找到 Python X.X.X
3. 右键 → 卸载
```

### 6.3 数据备份

#### 6.3.1 导出设计方案
```
卸载前建议备份：
1. data/ 目录
2. 所有 scheme_*.json 文件
3. 所有 exp_*.json 文件
```
